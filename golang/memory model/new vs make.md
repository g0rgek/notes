# `new(T)`

- Выделяет память под нулевое значение типа **T**
- Возвращает **`*T`** (указатель)
- Работает для **любой** структуры данных
- **Не** инициализирует внутренние структуры
- Полезно, когда нужен указатель на нулевое значение
Пример:
```go
p := new(int) // *p == 0`
```
---
# `make(T, ...)`

- Создаёт и **инициализирует** срезы, карты и каналы
- Возвращает **значение T**, а не указатель
- Даёт **ненулевой, готовый к использованию** объект
- Инициализирует внутренние структуры: массивы, хэш-таблицы, буферы каналов
Пример:
```go
s := make([]int, 0, 10) // рабочий срез
```
---
# Ключевые различия

| Характеристика | `new`                   | `make`                          |
| -------------- | ----------------------- | ------------------------------- |
| Работает с     | любым типом             | slice, map, chan                |
| Возвращает     | указатель (`*T`)        | значение (`T`)                  |
| Инициализация  | только нулевое значение | готовая структура данных        |
| Назначение     | получить указатель      | получить рабочий slice/map/chan |
# Быстрые ответы

- **Когда использовать `new`?** Когда нужен указатель на нулевое значение.
- **Когда использовать `make`?** При создании slice/map/chan для мгновенного использования.
- **Почему нельзя `make` для struct?** Структуры не требуют внутренней инициализации рантаймом.
- **Где выделяется память (`heap` или [stack](../memory%20model/Стек%20горутины.md))?** Решает escape-анализ, а не программист.
